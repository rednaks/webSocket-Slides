Pourquoi ?

> Les Applications Web qui requièrent une communication temps réel fiable :
>> Les Jeux en ligne
>> Les réseaux sociaux
>> Les applications financières
>> etc ...

Pourquoi ? Problème avec le HTTP ...

> À cause des limites du HTTP, c'est difficile de faire une application web qui communique en temps réel.

> Le HTTP est half-duplex (envoie de message dans une seule direction à la fois)

> Le HTTP est trop lent, et c'est chiant !

Pourquoi ? Simuler le temps réel ?

Quoi ?


C'est Quoi le WebSocket ?

> Une spécification du HTML5.
> Utilise le protocole WebSocket au lieux du HTTP
> Un cannal de communication full-duplex
> WebSocket sécurisé :
> > ws://
> > wss://
